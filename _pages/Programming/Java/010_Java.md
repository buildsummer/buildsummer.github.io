## 1. 자바 언어의 핵심 규칙과 구동 환경

### 파일 및 네이밍 규격
* **1 파일 = 1 Public 클래스**: 하나의 파일에 여러 클래스를 둘 수 있지만, `public` 클래스는 단 하나만 존재해야 하며 **파일명과 반드시 일치**해야 한다. (통상적으로 관리 편의상 1파일 1클래스가 권장된다.)
* **식별자 명명 규칙**:
  * **클래스/인터페이스**: 명사형, `PascalCase` (시작은 대문자, 각 단어 시작도 대문자)
  * **메서드/변수**: 동사형/명사형, `camelCase` (시작은 소문자, 이후 단어는 대문자)
  * **상수**: `UPPER_SNAKE_CASE` (전체 대문자, 언더바 `_` 연결)

### JVM 진입점과 표준 입출력
* **메인 메서드 구조**: JVM이 인스턴스 생성 없이 실행해야 하므로 반드시 `static`이어야 한다.
```java
public static void main(String[] args) {
    System.out.println("문자열끼리의 " + "연결도 가능합니다.");
    System.out.println("숫자" + 3 + "과 문자열의 연결도 가능합니다.");
}
```
* **표준 입출력 스트림**: `System` 클래스를 통해 제어한다.
  * `System.in` : 표준 입력 스트림 (키보드 등)
  * `System.out` : 표준 출력 스트림 (모니터 콘솔)
  * `System.err` : 표준 에러 출력 스트림

## 2. 데이터 타입 시스템과 메모리 가시성

자바의 자료형은 크게 **primitive type**과 **reference type**으로 나뉘며, 메모리가 할당되는 위치와 데이터 처리 방식이 완전히 다르다.

### primitive type (원시 자료형)
* `new` 키워드로 생성할 수 없으며 리터럴로만 값을 세팅한다. **Stack 메모리**에 실제 값이 직접 저장된다.
* 자바는 안전성을 위해 초기화되지 않은 값(특히 지역 변수)은 컴파일러 레벨에서 사용을 차단한다.
* **정수형**: `byte` (1 바이트), `short` (2 바이트), `int` (4 바이트), `long` (8 바이트, 뒤에 `L`/`l` 필수)
* **실수형**: `float` (4 바이트, 뒤에 `F`/`f` 필수), `double` (8 바이트)
* **기타**: `char` (2 바이트, 유니코드, 작은따옴표 `'c'` 사용), `boolean` (1 바이트, `true/false`)

### wrapper class & 메모리 누수 주의점
* primitive type을 객체화하여 Heap 영역에 띄우는 **immutable(불변) 클래스**이다. 인스턴스에 있는 값을 변경할 수 없다.
* 종류: `Byte`, `Short`, `Integer`, `Long`, `Double`, `Float`, `Character`, `Boolean`
* **boxing & unboxing**: primitive type과 wrapper class 간의 타입 변환을 의미하며, 현대 자바는 **autoboxing / autounboxing**을 지원한다.
* ⚠️ **실무 주의사항**: wrapper class는 동기화가 필요할 때 사용할 수 있으나, 반복문 내부에서 primitive type과 혼용하여 무분별한 대입이나 연산을 가하면 수많은 불필요한 객체가 생성되어 GC 부하를 유발하므로 연산 시에는 primitive type 사용을 권장한다.

### 안전한 타입 변환 (Casting)
* 데이터 손실이 발생하는 대입 연산은 허용하지 않는다. 표현 범위가 큰 타입에서 작은 타입으로 대입(바이트 크기가 작은 거로 변환)하면 데이터 손실 위험이 있어 컴파일러(`javac`)가 에러를 발생시킨다. 반드시 `(자료형)` 명시적 캐스팅을 표기해야 한다.
* 보통 각 클래스 내에 `parse`로 시작하는 클래스 메서드(`static method`)가 정의되어 있어 이를 이용해 형변환한다. (예: `Integer.parseInt()`)

### 변수 스코프와 초기화 블록
* **자동 초기화**: 클래스 변수(`static`)와 인스턴스 변수는 초기화를 생략하더라도 자바 시스템이 알아서 기본값(`0`, `false`, `null`)으로 자동 초기화해 주나, **지역 변수**는 초기화하지 않고 사용하려고 하면 컴파일 에러가 발생한다.
* **초기화 블록 (Initialization Block)**: 필드 변수마다 초기값을 세팅하지 않고 중괄호 블록 안에서 일괄 초기화하는 문법이다.
  * 인스턴스 초기화 블록: `{ 변수명 = 값; }` (인스턴스 생성 시마다 실행)
  * 클래스 초기화 블록: `static { 변수명 = 값; }` (클래스가 메모리에 로드될 때 단 한 번 실행)

## 3. 연산자와 흐름 제어의 특수 기법

### 연산자 특이점
* `a instanceof A`: 객체 `a`가 클래스/인터페이스 `A` 계통의 인스턴스가 맞는지 검증하여 `boolean`을 반환한다.
* 삼항 연산자: `조건식 ? 참일때값 : 거짓일때값`
* **매직 넘버 (Magic Number)**: 코드 내에서 상수로 선언되지 않고 의미가 모호한 채로 직접 하드코딩되어 사용되는 숫자들을 뜻하며, 가독성을 위해 상수로 치환해야 한다.

### 제어문 테크닉
* **라벨을 사용한 break**: 중첩 반복문을 한 번에 탈출하고 싶을 때 라벨을 지정하여 탈출 가능하다.
```java
allLoop :
for (int i = 2; i < 10; i++) {
    for (int j = 2; j < 10; j++) {
        if (i == 5) {
            break allLoop; // 외곽의 allLoop를 깨고 완전히 빠져나감
        }
        System.out.println(i + " * " + j + " = " + (i * j));
    }
}
```
* **enhanced for문**: 자바에서 따로 `foreach`라는 키워드를 쓰지는 않고 `for` 키워드를 그대로 활용한다. 인덱스 없이 순차 탐색할 때 가독성이 극대화된다.
  * 구조: `for (type 변수명 : iterate 대상) { 변수 사용; }`

## 4. 자바 객체 지향 프로그래밍 (OOP)의 본질

자바는 클래스 기반의 언어이다. **클래스(과자 틀)**를 기반으로 실제 메모리에 구현된 실체인 **인스턴스(=객체, 찍혀 나온 과자)**를 만든다. 실행 후 반환값이 없는 메서드에는 `void` 키워드를 붙인다.

### 자바의 유일한 규칙: Call by Value
* ⚠️ **동작 메커니즘**: 자바는 기술적으로 **항상 값에 의한 호출(Call by Value)로만 동작**한다.
* primitive type을 넘기면 변수의 실제 값만을 복사해서 전달하므로 호출된 메서드 내에서 값을 변경해도 외부 원본 변수는 변경되지 않는다.
* 객체(참조 타입)를 보낼 때도 객체 자체가 아니라 **'객체의 주소값'을 복사해서 넘기는 것**이다. 따라서 주소를 타고 들어가 내부 필드를 고치면 원본 객체의 상태가 바뀌지만, 메서드 안에서 파라미터 변수 자체에 `new` 객체를 새로 대입하더라도 외부 원본 변수의 참조 주소는 끊기지 않고 그대로 유지된다.

### 생성자 메커니즘
* 생성자가 아무것도 없으면 컴파일러가 매개변수가 없는 **default 생성자**(`Dog() {}`)를 알아서 만든다. 생성자를 명시했더라도 default 생성자가 필요하면 그냥 추가로 명시하면 된다.
* **`this()` 메서드**: 동일 클래스 내의 다른 생성자를 호출하는 것이며, 코드 중복을 제거한다. (반드시 생성자 첫 줄에 위치)
```java
Car() {
    this("소나타", 2012, "검정색", 160); // 다른 생성자를 호출함
}
```
* **`super()` 메서드**: 부모 클래스의 생성자를 호출할 때 사용한다. (반드시 생성자 첫 줄에 위치)

### Overloading vs Overriding
* 메서드 시그니처는 **매개변수의 타입, 순서, 개수**를 의미한다.
* **Overloading(오버로딩)**: 이름은 같지만 메서드 시그니처가 달라야 한다. (이름만 같고 완전 새로운 메서드를 만드는 것)
* **Overriding(오버라이딩)**: 부모 클래스의 메서드를 자식 클래스에서 동일한 이름 and 파라미터로 재정의하는 것이다. 상위 클래스에 있더라도 메서드 호출 시 최하위 자식의 오버라이딩 메서드가 최우선 순위로 호출된다(동적 바인딩).

## 5. 상속과 다형성, 그리고 설계 규격

### 클래스 상속 (`extends`)
* `class Dog extends Animals {}` // Dog 클래스가 Animal을 상속받음.
* **Is-A 관계**: 자식 클래스는 부모 클래스의 일종이다 (`Dog is an Animal`).
* **다중 상속 금지**: 자바는 클래스 간의 다중 상속을 지원하지 않는다.

### 다형성(Polymorphism)과 타입 캐스팅
* 하나의 객체가 여러 가지 타입을 가질 수 있는 성질로, 부모 타입 변수가 자식 인스턴스를 참조할 수 있다 (`Animal dog = new Dog();`).
* **업캐스팅**: 자식 타입을 부모 타입으로 바꿀 때는 자동 형변환된다.
* **다운캐스팅**: 부모 타입을 자식 타입으로 돌릴 때는 무조건 명시적 캐스팅 표기가 필수이다. 이때 실제 인스턴스 타입이 맞는지 확인하기 위해 `변수 instanceof 클래스명` 연산자를 사용해 검증하면 안전하다.

### Overriding의 3대 제약조건
1. 반환 타입은 부모 클래스 타입으로 안전하게 변환 가능한 범위(**공변 반환 타입**) 내에서 변경 가능하다.
2. 자식의 접근 제어자를 부모 클래스의 메서드보다 더 좁은 범위로 변경할 수 없다. (예: 부모가 `protected`면 자식은 `private` 불가능)
3. 부모 클래스의 메서드보다 더 넓고 큰 범위의 예외(Exception)를 자식 메서드가 마음대로 선언하여 던질 수 없다.

### 추상 클래스 vs 인터페이스
* **추상 클래스 (`abstract class`)**: 인스턴스 자체 생성이 불가하며, 자식 클래스가 부모의 추상 메서드를 필수로 상속받아 오버라이딩 구현하도록 강제한다. 선언부만 존재하고 구현체(몸체)는 없다. 추상 클래스 내에서는 `default` 키워드를 사용하지 않는다.
```java
abstract class p1 extends Animal {
    abstract void getfood(); // 추상 메서드 (세미콜론으로 끝냄)
}
```
* **인터페이스 (`interface`)**: 자바는 클래스 다중 상속을 지원하지 않는 대신, 인터페이스 다중 구현을 통해 다중 상속과 유사한 효과를 낸다. 규격에 의존하도록 설계하여 코드 간의 결합도를 낮추고 독립성을 유지하는 데 의의가 있다. 인터페이스는 클래스에 상속시킬 때 여러 개를 `implements` 할 수 있으며, 인터페이스가 또 다른 인터페이스를 상속할 때는 `extends`를 쓴다.
```java
접근제어자 interface p1 {}
접근제어자 interface p2 {}
class Dog extends Animals implements p1, p2 {} // 다중 구현
void ftn(p1 P) {} // 인터페이스를 매개변수 타입으로 활용
```
  * **구성 요소 제한**: 오직 추상 메서드와 상수만 포함 가능하다. 모든 메서드는 `public abstract`이며, 모든 상수는 **`public static final`**이 생략된 형태로 적용된다.
  * **디폴트 메서드 (`default`)**: Java 8부터 지원하며, 인터페이스 내부임에도 `default` 지시어를 붙이면 몸체(`{}`)를 직접 구현할 수 있습니다. 구현 클래스들이 매번 일일이 오버라이딩하지 않아도 공통으로 사용 가능하다.

## 6. 제어자와 패키지 시스템 (캡슐화)

### 접근 제어자 범위
* `private` (해당 클래스 내부만) $\rightarrow$ `default` (생략 시 기본값, 동일 패키지 내부만) $\rightarrow$ `protected` (동일 패키지 + 패키지가 다르더라도 상속받은 자식 클래스면 허용) $\rightarrow$ `public` (제한 없음)
* **특징**: 외부에서 직접 접근할 수 없는 `private` 변수라도 멤버 메서드가 `public`이면 그 메서드를 경유하여 변수 접근 및 수정이 가능하다. `default` 접근 제어자는 패키지가 다르면 인스턴스를 생성했다 하더라도 변수 직접 접근이 불가능하다.

### 기타 제어자 기법
* **`final`**: '최종'의 의미를 가지며 한 번 설정하면 수정 불가함을 의미한다. (C/C++의 `const`와 유사하나 자바에서 `const`는 기능이 없는 예약어이다.)
  * 변수: 값 수정 불가 (상수화). 관례적으로 전체 대문자와 언더바(`_`) 조합으로 명명하며 주로 `static final` 조합으로 구현한다. 단, 객체 변수에 `final`을 쓸 경우 **재할당이 안 될 뿐이지 객체 내부의 데이터를 변경하거나 컬렉션에 요소를 `add`, `remove` 하는 행위는 가능하다.**
  * 메서드: Overriding 불가.
  * 클래스: 상속 불가.
* **`static`**: '공통적인', '클래스 레벨의' 의미를 가진다. 인스턴스 변수/메서드를 클래스 변수/메서드로 변환한다. 프로그램 시작 시(클래스 로딩 시) 메모리에 단 한 번만 생성되고 초기화되며, 인스턴스를 생성하지 않아도 클래스명으로 바로 사용 가능하다.
* **`static 메서드`**: 클래스를 통해 직접 호출한다. 메서드 내부에서 인스턴스 멤버 변수에는 접근이 불가능하다. 상태를 갖지 않고 연산만 수행하는 유틸성 메서드(`Util`)를 만들 때 프로시저처럼 재사용하기 위해 주로 쓰인다.
* **`abstract`**: '추상적인' 의미를 가지며 클래스와 메서드에 사용 가능하다.

### 패키징과 임포트
* 패키지는 물리적으로 디렉토리(폴더) 구조를 의미하며 모든 클래스는 반드시 하나 이상의 패키지에 포함되어야 한다. 선언이 없는 경우 default 패키지(`unnamed package`)에 포함된다.
```java
package 패키지명;
package 패키지명.sub패키지;
```
* 호출 및 사용은 `import 패키지명.sub.*;` 형태로 한다. `*`를 사용하여 임포트할 때는 해당 경로 직하에 있는 클래스들만 포함될 뿐, 그 하위 디렉토리(서브 패키지) 내부의 클래스까지 자동으로 임포트되지는 않는다.
* `java.lang` 패키지는 예외적으로 자바가 자동으로 임포트해주므로 문구 작성이 필요 없다.

## 7. 자바 API 양대 산맥 (`java.lang` & `java.util`)

### `java.lang` 패키지: 불변과 가변
* **`Object`**: 모든 자바 클래스의 최상위 조상 클래스이다. **기본적으로 딱 11개의 메서드만으로 구현**되어 있으며, 모든 객체는 이 메서드들을 상속받는다.
  * `toString()` : 객체 정보 문자열 출력
  * `equals()` : 객체 비교 (각 API 하위 클래스 목적에 맞게 Overriding하여 재정의함)
  * `hashCode()` : 객체 식별값 반환
  * `clone()` : 인스턴스 복사 기능 (`Cloneable` 인터페이스를 명시적으로 구현한 클래스의 인스턴스에서만 작동함)
  * 이 외에 스레드 제어를 위한 `wait()`, `notify()`, `notifyAll()` 등 총 11개의 메서드가 존재한다.
* **`String`**: 대표적인 **immutable(불변) 객체**이다. 한 번 생성되면 바뀔 수 없으므로 `equals()`, `indexOf()` (인덱스는 0부터 시작), `contains()`, `charAt()`, `replaceAll()`, `substring()` 등의 연산들은 모두 **새로운 객체를 생성하여 return**하는 것이다.
* **`StringBuffer` vs `StringBuilder`**: 하나의 객체를 계속 변경하는 **mutable(가변) 객체**들이다. (`append()`, `insert()`, `delete()` 지원)
  * `StringBuffer`: 멀티스레드 환경에서 안전하도록 **동기화(Synchronization)를 지원**하지만 버퍼 오버헤드가 있다. `toString()`으로 String 변환이 가능하다.
  * `StringBuilder`: StringBuffer와 기능은 같으나 동기화를 지원하지 않는 대신 **성능이 우수**하다.

### `java.util` 패키지: 컬렉션과 정렬
* **배열 (Array)**: 자료형 옆에 `[]`를 붙여서 생성한다. primitive type도 배열로 선언하면 배열 자체는 클래스(객체)이다. 예시로 `int[] grade = new int[]{85, 65, 90};` 형식을 쓰며 `grade.length`로 길이를 확인한다. 
  * **Arrays 클래스**: 배열을 정렬, 복사, 검색하는 연산을 지원하는 유틸리티 클래스이며 모든 메서드가 `static`이다.
* **컬렉션 프레임워크 (Collection Framework)**: 배열은 길이가 정해져 있지만 리스트는 크기가 변하는 동적 구조이다.
  * **계층 구조**: `Map` 인터페이스만 별개의 독립 구조를 가지며 기본 계층은 `Iterable` $\rightarrow$ `Collection` $\rightarrow$ `List`, `Set`, `Queue` 구조이다.
  * **ArrayList 예시**: `ArrayList<String> pitches = new ArrayList<>();` (객체를 포함하는 리스트는 타입을 명기하는 제네릭스 사용을 권고한다.)
  * **Map**: Key-Value(키-값) 쌍으로 이루어진 클래스이다. 예시: `HashMap<String, String> map = new HashMap<>();`
* **이터레이터 (Iterator)**: 컬렉션 요소를 순차적으로 순회하며 꺼내오기 위한 표준 인터페이스이다. 다만 현대 자바 코드에서는 가독성 면에서 이 구문보다 **enhanced for문** 사용을 더 권장한다.
```java
Iterator<Integer> iter = lnkList.iterator();
while (iter.hasNext()) { System.out.print(iter.next() + " "); }
```
* **정렬 인터페이스 (Comparable / Comparator)**:
  * `Comparable`: 클래스의 기본 정렬 기준을 세울 때 사용한다. `compareTo` 메서드를 구현해야 하며, 대개 오름차순으로 기본 정렬된다.
  * `Comparator`: 기본 오름차순 외에 내림차순이나 사용자 정의 정렬 등 특별한 정렬 기준으로 요소를 정렬하고 싶을 때 따로 구현하여 주입한다.

## 8. 제네릭스(Generics)와 타입 안정성

### 컴파일 타임의 수호자
* 데이터 타입을 일반화하는 것을 의미한다. 클래스나 메서드 내에서 사용할 타입을 컴파일 시에 미리 지정하는 것으로, `<>`로 객체 타입을 명기한다.
* 제네릭스가 없으면 자료형이 암묵적으로 `Object`로 캐스팅되어 저장되기 때문에 데이터를 가져올 때 항상 캐스팅(형변환)을 해줘야 하는 불편함이 있지만, 명시하면 자료형 강제 변경을 막고 타입 안정성을 얻는다.
* primitive type은 제네릭 타입으로 사용할 수 없기 때문에 wrapper class를 사용해야 한다.
* **타입 소거(Type Erasure)**: 컴파일 타임에 전부 타입이 체크 및 변환되어 컴파일된 파일(`.class`)에는 제네릭 정보가 남지 않는다.
```java
class MyArray<T> { T element; }
MyArray<Integer> myArr = new MyArray<>(); // Java SE 7부터 우변 생략 가능.
```
* `<T extends LandAnimal>`과 같이 선언하면 특정 타입만 사용 가능하도록 타입 제한이 가능하다. 인터페이스로 제한할 때도 `implements`를 쓰지 않고 `extends` 키워드를 써야 하며, 동시에 여러 제한을 걸 때는 `&` 기호로 연결 가능하다.

### 와일드카드 문법
* `<?>` : 모든 타입 사용 가능
* `<? extends T>` : T와 그 하위 타입만 사용 가능
* `<? super T>` : T와 그 상위 타입만 사용 가능

### 제네릭 메서드
* 클래스에 선언된 `T`와는 완전히 독립적인 별개의 `T`를 가진다. 오버로딩 없이 여러 타입을 유연하게 받을 수 있습니다.
```java
public static <T> T printAndReturn( ... ) { ... }
box.<String>printAndReturn("Hello");
```

## 9. 모던 자바: 고급 함수형 인터페이스와 익명 구조

### 익명 구조와 람다
* **내부 클래스 (Inner Class)**: 클래스 안에 또 다른 클래스를 선언하는 구조로 외부 클래스의 멤버에 제약 없이 접근 가능하다. 외부에서는 내부 클래스에 접근하기 까다로워지므로 캡슐화가 증가한다.
* **익명 클래스 (Anonymous Class)**: 클래스 선언과 인스턴스 생성을 동시에 진행하며 이름이 존재하지 않는 단 하나의 일회성 객체용 클래스이다. 람다 표현식의 근간이 된다.
```java
new Object() { int min(int x, int y) { return x < y ? x : y; } }
```
* **람다 (Lambda)**: `(매개변수) -> { 몸체 }` 형태로 코드를 전달한다. 클래스 선언 없이도 함수를 객체처럼 다루며 익명 객체를 간결하게 생성 가능하다. 단, 인터페이스 내부에 추상 메서드가 단 1개인 경우 (`@FunctionalInterface`)에만 람다 함수를 사용할 수 있습니다.
* **메서드 참조 (Method Reference)**: 람다 표현식이 단 하나의 메서드만 호출하며 아규먼트를 그대로 전달하는 구조일 때 축약하는 기법이다. (`(base, exponent) -> Math.pow(base, exponent);` $\rightarrow$ `Math::pow;`)

### 가공과 소비의 스트림 (Stream API)
* 데이터 소스의 컬렉션이나 배열 이터레이션(반복문) 과정을 단순화 및 추상화해 준다. 스트림 객체를 생성하면 내부 구조를 통해 요소를 하나씩 알아서 호출한다.
```java
Stream<Integer> stream = list.stream();
stream.forEach(System.out::println);
```
* **중개 연산**: 스트림 흐름 중간에서 데이터를 가공 및 필터링하며 결과로 다시 새로운 스트림을 반환한다. 최종 연산이 호출되기 전까지는 중간 연산이 실제로 수행되지 않고 대기하는 **지연 연산(Lazy)** 특징을 가진다. (`stream.distinct().filter(n -> n % 2 != 0).sorted()`)
* **최종 연산**: 지연 되었던 앞선 모든 중개 연산들을 한 번에 프로세싱하며 스트림을 소비하고 끝낸다. (`.forEach(System.out::println);`)

### `Optional<T>`의 도입 목적
* 객체를 다룰 때 발생할 수 있는 `NullPointerException`을 방지하기 위해, 일반 참조형 객체를 한 번 더 감싸서 안전하게 넘겨주는 데이터 래퍼 클래스이다. (`Optional.of(A)` 또는 `Optional.ofNullable(A)`)

## 10. 엔터프라이즈의 핵심: 예외 처리와 멀티스레딩

### 예외 처리 아키텍처
자바의 모든 예외 객체는 `Throwable` 클래스를 기반으로 하며, 시스템 다운을 일으키는 `Error`와 코드단에서 수습 가능한 `Exception`으로 분리된다.

```java
try { // 다중 캐치 블록 및 멀티 캐치(catch (errorClass1 | errorClass2 e1)) 지원
} catch (errorClass1 e1) {
} finally { // 예외 발생 여부와 상관없이 무조건 실행 }
```
* **try-with-resources**: 사용이 끝나면 자원을 자동으로 반환하고 해제해 주는 문법 구조이다. `try` 옆 괄호 안에 자원 할당 코드를 넣으면 블록이 종료될 때 자동으로 Close 된다. (`try (자원할당) {}`)
* **사용자 정의 예외**: `Exception` 클래스도 결국 일반 클래스이므로 상속을 받아 커스텀 에러 예외 클래스 생성이 가능하며 `throw`를 이용해 임의로 발생시킬 수ust 있다.
* **예외의 종류**: `RuntimeException` (런타임 예외, 컴파일러가 체크 안 함)과 이외의 예외 (Checked Exception)로 나뉜다. 런타임 익셉션 계열이 아닌 예외들은 컴파일러가 `try-catch`나 `throws` 예외 전가를 강제하고 있다.
```java
class a {
    void method() throws Exception { // 예외 던지기 명시
        throw new Exception("오류메세지"); // 예외 직접 발생
    }
}
```

### 멀티스레드와 동시성 제어
* **프로세스와 스레드**: 하나의 프로세스는 실행에 필요한 데이터, 메모리, 그리고 실제 작업을 수행하는 스레드들로 구성된다. 하나의 프로세스는 반드시 하나 이상의 스레드를 가진다.
* **문맥 교환 (Context Switching)**: 물리적인 CPU 코어 개수보다 구동 중인 스레개수가 더 많아지면 코어가 스레드들을 번갈아 가며 시분할 처리한다. 이때 스레드가 교체되는 과정에서 문맥 교환 비용이 발생하여 잦아질 경우 전체 효율이 저하될 수 있다.
* **자바 스레드 생성 방법 2가지**:
  1. `Runnable` 인터페이스 구현: 자바는 클래스 다중 상속을 허용하지 않으므로 `Thread` 클래스를 상속받으면 다른 클래스를 상속받을 수 없게 된다. 따라서 대개는 확장성이 좋은 인터페이스 구현 방식을 주로 사용한다.
  2. `Thread` 클래스 직접 상속
* **스레드 그룹 및 데몬 스레드**: 스레드 그룹을 통해 여러 스레드를 묶어서 한 번에 관리할 수 있다. **데몬 스레드 (Daemon Thread)**는 주 스레드의 작업을 보조하는 백그라운드 스레드로, 주 스레드가 종료되면 데몬 스레드도 작업 완료 여부와 상관없이 강제 자동 종료된다. 대표적인 예가 JVM의 **가비지 컬렉터(Garbage Collector)**이다.
